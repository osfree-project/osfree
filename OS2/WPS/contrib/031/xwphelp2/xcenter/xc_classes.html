De "Classes" pagina in een &xcenter; instellingen notitieblok biedt u de mogelijkheid op te kijken naar de widget classes die momenteel beschikbaar zijn.

Let op dat widget classes geen echte WPS classes zijn. Alleen het &xcenter; gebruikt de class namen om de widgets intern te organiseren. Beschouw een widget class als een model waarvan widgets gemaakt kunnen worden. Een widget class is ofwel ingebouwd in het &xcenter; zelf of geladen vanuit een plug-in DLL vanuit de plugins\xcenter directory. Zo'n plug-in DLL kan een of meer widget classes bevatten.

In tegenstelling tot de "Widgets" pagina, welke de huidige widgets laat zien van een enkele &xcenter;, laat de "Classes" pagina dezelfde widget classes zien voor elke &xcenter;. De classes die hier getoond worden worden dan ook getoond in het "Voeg widget toe" submenu van een open &xcenter; zijn &popmenu;.

De volgende velden worden getoond:

  • Het "DLL" veld toont de naam van de dynamische link library (DLL) welke de widget implementeerde. Dit is ofwel een van de DLL's in de plugins\xcenter directory ofwel zal het "Ingebouwd" weergeven als het widget ingebouwd is in &xwp; zelf.

  • De "Widget class" laat de interne naam van de class zien die gebruikt wordt door het &xcenter; om de class te identificeren waar een widget bij hoort. Deze naam is kritisch voor de werking van een widget en zal nooit veranderen, tenzij de DLL niet langer beschikbaar is.

  • Het "Class naam" veld is hetzelfde als in het "Voeg widget toe" menu en is een beschrijvende naam van de class. Dit is alleen voor informatieve doeleinden en niet kritisch voor de werken van het widget.

  • Het "Versie" veld toont de minimale &xcenter; versie dat het widget nodig heeft om te werken. Het &xcenter; controleert dit versie nummer om er zeker van te zijn dat het de plug-in DLL kan laden. Dit veld is niet van betekenis voor ingebouwde widgets en is daarom leeg voor die widgets.
