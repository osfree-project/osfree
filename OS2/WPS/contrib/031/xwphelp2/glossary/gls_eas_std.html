&os2; stelt zelf een aantal uitgebreide kenmerken (EA's) vast. Deze worden "standaard EA's" genoemd. De veelvuldig gebruikte standaard EA's zijn:
  • .CLASSINFO wordt door de Workplace Shell gebruikt om zijn object specifieke gegevens met bestandssysteem objecten op te slaan. Dit zijn binaire gegevens die zaken bevatten als de naam van de class van het object en de meeste van de instellingen als die zijn gewijzigd in de standaard instellingen in het instellingen notitieblok van de objecten.

  • .COMMENTS, .KEYWORDS, en .SUBJECT worden gebruikt om extra informatie over het bestandssysteem object op te slaan en komen overeen met de respectievelijke velden op de "Bestand" pagina van het instellingen notitieblok van de objecten.

  • .ICON wordt gebruikt als een bestandssysteem object wordt toegewezen als een niet-standaard pictogram, dat dan ook in de Workplace Shell zal worden getoond. Als u het pictogram van een bestandssysteem object wijzigt op de "Pictogram" pagina van zijn instellingen notitieblok, wordt dit pictogram opgeslagen in deze EA. (Voor mappen, als toevoeging, .ICON1 bevat het "open map" icon van de map.)

  • .LONGNAME wordt door de Workplace Shell gebruikt om een naam van een object in op te slaan, als het niet overeenkomt met de naams conventies van het bestands systeem waar het object toe behoort. Zie "Echte namen" voor details.

    LI>.TYPE bevat het bestandstype van het object als het verandert is van de standaard in zijn instellingen notitieblok.
