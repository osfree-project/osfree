Om de instellingen van een object te kunnen wijzigen van "buiten" de Workplace Shell, ondersteunt de WPS het concept van object installatie reeksen.

Een WPS installatie reeks is een opeenvolging van steekwoord=waarde paren, gescheiden door een punt-komma ';'. De steekwoorden die door een object worden begrepen zijn afhankelijk van zijn class.

B.v., om van een map de gesloten en open pictogrammen te wijzigen, kunt u de volgende tekenreeks gebruiken:

ICONFILE=C:\closed.ico;ICONNFILE=1,C:\open.ico;

Omdat de WPS classes gegroepeerd zijn in een hi‰rarchische boomstructuur en van elkaar overerven, zal een object niet alleen de installatie reeks van zijn eigen class begrijpen maar ook die van al zijn ouder classes. B.v., een map zal niet alleen WPFolder steekwoorden begrijpen, maar ook de algemene WPObject steekwoorden (die door alle objecten worden begrepen).

Er zijn zeer veel verschillende object steekwoorden voor de verschillende WPS classes. Selecteer uit de volgende lijst om er iets over le leren:

  • WPObject (alle objecten)
  • WPFolder (alle mappen)
  • WPDesktop (de Werkplek)
  • XWPTrashCan (de prullenbak)
  • WPProgram (alle programma objecten)
  • XCenter (het &xcenter;)
  • XWPString (installatie reeks objecten)
Er zijn verschillende manieren om installatie reeksen toe te passen op een object.
  • &xwp; staat U toe om installatie reeks objecten te gebruiken.

  • Uit REXX, kunt U de functie SysSetObjectData gebruiken of, als U een nieuw object aan het maken bent, de functie SysCreateObject.

  • Uit de C code, kunt U de PM WinSetObjectData en WinCreateObject API's * gebruiken.

  • Vanuit een WPS class, kunt U de WPObject::wpSetup methode, * gebruiken die veel subclasses te niet doet en de werkelijke installatie reeks verwerking implementeert binnen de WPS.
Opmerking: Gesteld tegenover een populaire misvatting, als een installatie reeks wordt aangeroepen door een object, wordt de reeks zelf nergens opgeslagen. In plaats daarvan wordt de reeks verwerkt door de WPS class van het object, vertaald in de binaire gegevens van het object, en dan verwijderd.

Met als resultaat dat, om in staat te zijn de installatie reeksen weer op te zoeken voor het tonen op de "Pictogram" pagina in een object zijn instellingen notitieblok, moet &xwp; de binaire gegevens van een object evalueren en juiste beslissing nemen wat de installatie reeks zou kunnen zijn die die binaire instelling zou moeten maken. Iets van de object gegevens is totaal niet beschreven en ondanks dat er een weg beschreven is om er achter te komen wat de binaire object gegevens betekenen, is &xwp; niet in staat om van daaruit een installatie reeks te maken
