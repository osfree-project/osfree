Het "WPS Class List" object toont u de complete WPS class list als een hi‰rarchische boomstructuur, met zeer veel additionele WPS en SOM informatie. Door dit object te gebruiken, kunt u zowel WPS classes registreren/deregistreren en vervangen en vervanging van classes ongedaan maken, die reeds zijn geregistreerd.

Zie "WPS classes" voor een introductie wat WPS classes inhouden.

Waarschuwing: Het WPS Class List object maakt het u mogelijk wijzigingen aan te brengen die uw systeem onbruikbaar maken. Als u de betekenis van een bepaalde class niet weet, of als u niet weet waar WPS classes allemaal voor dienen, moet u hierin geen wijzigingen aanbrengen. U kunt veilig alle informatie bekijken, maar u moet de WPS class list niet bewerken indien u niet precies weet wat u doet. Veel wijzigingen die deze pagina mogelijk maakt kunnen uw systeem instabiel maken (als u geluk hebt) of maken het starten van de Workplace Shell geheel onmogelijk zodat u misschien &os2; geheel opnieuw moet installeren. Zorg er in ieder geval voordat u een backup van de WPS maakt voordat ook maar iets gaat wijzigen.

Het class list venster is gesplitst in drie sub-vensters, waarvan de scheidingen d.m.v. de muis verplaatst kunnen worden.

Als u een class selecteert in de class boom in het linker venster, tonen de twee sub-vensters rechts additionele informatie over deze class.

  1. Rechtsboven, vindt u WPS gerelateerde informatie, zoals naam van de class, de standaard class titel (zoals getoond in de derde kolom van een gedetailleerde kijk op een map), en de DLL die de class bevat.

    Als deze DLL in het LIBPATH staat, is in de meeste gevallen geen extra informatie te zien. Dit is speciaal van toepassing voor de meeste standaard WPS classes, die zijn opgeslagen in \OS2\DLL\PMWP.DLL.

    Ook is het "class default" pictogram te zien, dat gewoonlijk 'hard-coded' in de class zit. Dit is het pictogram dat standaard wordt afgebeeld indien geen eigen pictogram is aangemaakt voor een object op de "Pictogram" notitieblok pagina.

    In de meeste gevallen, als de geselecteerde class bekend is bij &xwp; zijn maker, ontvangt u ook een korte beschrijving van de class.

  2. Rechts onder, wordt veel methode informatie getoond die direct uit de SOM kernel wordt gehaald (waarop de Workplace Shell afgaat om zijn classes te verenigen). Deze informatie is ongetwijfeld alleen van belang voor WPS programmeurs en praktisch gelijk aan de output van de befaamde "RWX" hulpprogramma.

    Afhankelijk van de bovenin gemaakte selectie, laat dit sub-venster de instantie methodes van de class methodes van de zojuist geselecteerde class zien.

    Voor iedere methode, krijgt u de volgende methode informatie (het kan zijn dat u de scheidingsbalk van de verzamelplaats moet verplaatsen om alle velden te kunnen zien):

    1. Methode index. Dit wordt intern door SOM gebruikt om snel methodes overeenstemmend met de methode tabellen te verklaren. Dit is de index waarin de methode is opgeslagen in de class methode tabel. Bedenk dat dezelfde methode niet noodzakelijk dezelfde index heeft in verschillende classes.

    2. Methode naam. Dat is dus de naam van de methode.

    3. Adres. De procedure pointer die SOM heeft teruggegeven toen de methode werd ingevuld. Deze pointer is natuurlijk afwijkend voor verschillende classes als een class een methode heeft opgeheven.

    4. Class. De class die de methode introduceerde (het eerst definieerde).

    5. Overschreven door. Een lijst van classes die deze methode hebben overschreven. Bedenk dat alleen classes tot de huidige geselecteerde class worden getoond. Als andere overge‰rfde classes van de huidige class deze methode overschrijven, wordt dat niet getoond. Selecteer de overerfde classes om die informatie te zien.

    Bedenk dat u de methode informatie kunt sorteren op vier verschillende criteria voor extra duidelijkheid. Klik rechts in het titel gebied van het methode scherm (waar de kolom titels staan), en u krijgt een &popmenu; met de sorteer opties.
U kunt de WPS class lijst wijzigen door &popmenu;s te gebruiken. Druk met muistoets 2 op iedere class naam in de class boom, en er zal een &popmenu; verschijnen, die u de aanwezige opties voor deze class toont. Bepaalde opties worden uitgeschakeld als &xwp; denkt dat u bepaalde wijzigingen niet moet maken.

Als u muis toets 2 in een leeg gebied van de linker window indrukt, verschijnt een ander &popmenu;. Dit geeft u de mogelijkheid om een nieuwe class te registreren .
