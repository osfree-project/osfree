Telkens als &xwp; een programma object start vanuit de &cfgfdr; (dat wil zeggen, wanneer u het betreffende menu onderdeel selecteert in een map &popmenu;), kan het een aantal trucjes met ze doen.

Deze trucjes zijn de volgende:

  1. Als de opstart directory van het programma object leeg is, zal &xwp; tijdelijk de directory van de map waarvan het &popmenu; gekozen is nemen voordat het programma echt opgestart wordt. Dit betekent dat u het zelfde programma van alle map &popmenu;'s kunt starten, maar met een variabele opstart directory, zijnde de directory van de map waarvan het &popmenu; is gebruikt.

    Bijvoorbeeld, met de standaard configuratie hebben de opdrachtregel objecten lege opstart directories. Dit is waarom u opdrachtregels kunt starten "in" de map waarvan het &popmenu; is gebruikt.

    Als u dit gedrag wilt uitschakelen, geeft u simpelweg een opstart directory op in een programma object (b.v. "C:\"), en &xwp; zal het met rust laten. In plaats daarvan zal het altijd de opstart directory gebruiken die u heeft opgegeven.

  2. Als de parameter lijst van een programma object geen afsluitend "%" teken bevat, zal de directory van de door u gebruikte map gebruikt worden om door te geven aan het programma.

    Dit biedt u de mogelijkheid om een volledig gekwalificeerde directory naam door te geven als een argument aan een programma. U kunt deze mogelijkheid volledig uitschakelen op de "&popmenu;'s" pagina van het "Workplace Shell" object.

    Als u dit gedrag voor slechts ‚‚n menu onderdeel uitgeschakeld wilt hebben (niet alle programma's kunnen overweg met directories als parameters), voegt u een "%" teken toe aan de "Parameters" lijst van het betreffende programma object. &xwp; zal het doorgeven van de extra parameter achterwege laten. Dit is bijvoorbeeld het geval met de opdrachtregel objecten van de standaard configuratie.

  3. U kunt de huidige inhoud van het klembord toevoegen aan de parameter lijst van het klembord door een "%**C" teken in de parameter lijst te stoppen. Dit kan op elke plek voorkomen in de parameters. Dit is hoofdlettergevoelig; "%**c" (in kleine letters) zal niet werken.

    Voorbeeld: Pad en bestandsnaam = "e.exe"; parameters = "%**C" zal de systeem editor opstarten en de inhoud van het klembord beschouwen als bestandsnaam.

    Let op dat de inhoud van het klembord afgekapt zal worden zodat de maximale lengte van de parameter lijst niet overschreden zal worden. Met &os2; is de maximale pad lengte 260

  4. Als de programma naam een "~" karakter bevat, zal &xwp; die tijdelijk verwijderen voordat het programma gestart wordt. U kunt dit gedrag ook uitschakelen in het "Workplace Shell" object.
  5. U kunt nu scheidingstekens toevoegen in een menu door "---" (drie koppelstreepjes) op te geven in de naam van een programma object. (Een scheidingsteken is een horizontale streep om optisch groepen van menu onderdelen te splitsen.) In dit geval zal &xwp; de instellingen van het programma object negeren (zoals het uitvoerbare programma, parameters enz.) en simpelweg een scheidingsteken toevoegen aan het menu. Dit werkt zowel voor het hoofd &popmenu; alsook voor de submenu's.
&xwp; implementeert de mogelijkheden 1.-4. door in werkelijkheid de instellingen van het programma object voor een fractie van een seconde aan te passen, het programma object te openen, en dan de instellingen terug te zetten naar de originele waarden.
