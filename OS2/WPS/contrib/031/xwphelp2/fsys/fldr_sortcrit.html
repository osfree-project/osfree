De volgende vier sorteer criteria zijn altijd beschikbaar in alle &link_folders;:
  • Sorteren op naam: dit sorteert een map alfabetisch overeenkomstig de titels van het object.

  • Sorteren op type: dit sorteert een map overeenkomstig de object typen, d.w.z. het .TYPE Uitgebreid Kenmerk dat kan worden ingesteld in het instellingen notitieblok van een object. Merk op dat dit alleen geldt voor objecten van data bestanden; alle andere objecten (en objecten van data bestanden van ongedefinieerd type) zullen aflopend worden gesorteerd.

  • Sorteren op class: dit sorteert de map overeenkomstig de object classes, d.w.z. dat wat zichtbaar is in de derde kolom van een Gedetailleerd overzicht van een map.

  • Sorteren op bestandsnaam uitbreiding: dit sorteert een map overeenkomstig de karakters na de laatste punt in de titel van een object, dat normaliter een bestandsnaam uitbreiding is (zoals .TXT, .EXE enz.). Objecten zonder een uitbreiding komen eerst.
Tenzij een map zijn sorteer class heeft gewijzigd naar iets anders dan het WSFileSystem, dat normaliter alleen maar gebeurt met speciale mappen zoals de vuilnisbak en de "Lettertypen" map, zijn de volgende sorteer criteria eveneens beschikbaar:
  • Sorteren op echte naam: dit sorteert een map alfabetisch overeenkomstig de bestandssysteem namen. Omdat alleen bestandssysteem objecten echte namen hebben, zullen onbenoemde objecten niet juist worden gesorteerd.

  • Sorteren op grootte: dit sorteert een map overeenkomstig de grootte van de bestanden. Omdat onbenoemde objecten geen bestandsgrootte hebben, maar alleen wat ruimte innemen in OS2.INI, wordt aangenomen dat zij een grootte hebben van nul bytes.

  • Sorteren op laatste schrijf / laatste toegangs / gemaakt op datum: dit sorteert de map overeenkomstig de bestandssysteem informatie, die ook niet beschikbaar is voor onbenoemde objecten. Merk op dat het FAT bestandssysteem al deze informatie niet ondersteunt, dus deze drie criteria zullen daar voor geen verschil zorgen.
Als de "Mappen eerst" instelling ook is ingeschakeld, worden alle mappen aan de bovenzijde van de map geplaatst.

In de groep mappen en objecten die geen mappen zijn, wordt het sorteren respectievelijk overeenkomstig de huidige sorteer criteria gedaan. B.v., als "sorteren op laatste schrijfdatum" wordt geselecteerd en "mappen eerst" is ook geselecteerd, worden alle submappen bovenin geplaatst, en alle submappen worden dan gesorteerd op laatste schrijfdatum, en alle objecten die geen map zijn worden ook op laatste schrijfdatum gesorteerd.
