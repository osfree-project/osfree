&xwp; offers you two special new folders for system startup and shutdown.

The &xwp; Startup Folder is a replacement of the standard &os2; startup folder. The original continues to work, and both folders can be used together.

Compared to the Desktop startup folder, the &xwp; startup folders have the following advantages:

  • You can have several startup folders -- the original Desktop startup folder, and any number of &xwp; startup folders.

  • For each of the &xwp; startup folders, you can set individually whether its objects should always be restarted or only after a reboot in its respective properties notebook.

  • The startup folders can display a progress window while processing the folder's contents, and you can cancel processing.

  • You can set initial and per-object delays for the startup folders in a startup folder's properties notebook.
By contrast, the &xwp; Shutdown Folder will be executed before the eXtended Shutdown procedure of &xwp;. By using this folder, you may have certain tasks executed which you need before switching off your computer.

For a detailed description of these two folders please consult the &xwp; User Guide.
