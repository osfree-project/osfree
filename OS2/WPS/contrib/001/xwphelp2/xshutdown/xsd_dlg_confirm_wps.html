In the confirmation window (which is displayed if you have enabled confirmations in the Desktop's properties notebook) the following settings are available:
  • You may determine whether &xwp; should close all sessions before restarting the WPS. If so, &xwp; will not only close the WPS windows, but all other programs also.

  • You may also determine whether after the Desktop restart &xwp; should process all startup folders again. If this is enabled, &xwp; will process all &xwp; startup folders after the Desktop restart. By contrast, if this setting is disabled, only those startup folders will be processed which have the "Every Desktop restart" setting enabled in their respective properties notebook.

    Note: You should only enable this setting if you have either "Close all sessions" enabled also or if you're sure that the programs in the startup folder are not currently running, because otherwise they might be started twice.

    This setting does not affect the regular &os2; startup folder. That folder is never processed after a Desktop restart.

  • If you select "Archive desktop once after restart", &xwp; will enable the "Archiving" feature once before the restart and disable it again after the Desktop has restarted. This is just a shortcut to manually enabling the archiving settings in the Desktop's properties notebook and then disabling them again.

  • If you deselect the "Show this message next time" setting, confirmations will be disabled for eXtended Shutdown and Desktop restarts. You can then re-enable the confirmations in the Desktop's properties notebook on the "&xshutdown;" page.
