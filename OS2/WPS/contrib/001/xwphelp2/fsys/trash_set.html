This notebook page allows you to fine-tune the behavior of the &xwp; trash can.

Press "Undo" to restore the settings that were in use before you opened this notebook page.

Press "Default" to restore the settings to the installation state.

If "Confirm empty" is checked, you will get a chance to confirm emptying the trash can.

If "Confirm destroy" is checked, you will get a confirmation window when you select "Destroy object" of a trash object's &popmenu;.

If "Always delete immediately" is checked, &xwp; will perform all delete operations directly, without moving the objects into the trash can first. In that case though, you will not be able to recover objects that were accidentally deleted. Note that you can perform an "immediate delete" for specific objects even if this setting is not enabled; see "Deleting objects" for details. -- This setting corresponds to the "Replace true delete" feature that was previously only on the "Features" page in the "&xwp; Setup" object.

Note that if you are using &xwp;'s eXtended Shutdown, you can also have the trash can emptied automatically before the system is shut down. This can be enabled in the Desktop's properties notebook.
