This window appears when you copy, move, or rename a file and a file with the same real name already exists in the target folder. This applies to folders and data files.

&xwp; replaces the default &os2; dialog for this situation with a new, improved one, which gives you additional information and additional options. &xwp; also remembers the last selection you made, which will automatically be selected the next time this window appears. In addition, the position of this window is saved, so that you may now determine the position of this confirmation dialog.

Select "Replace existing object" to have the existing object deleted and replaced by the new object.

Select "Rename new object" to rename the object that you were just working on to something else. &xwp; proposes you a new title for this object so that the corresponding real name will not conflict with other objects in the folder.

Select "Rename existing object" to rename the object which already exists in the target folder instead.

Press "OK" to finish the operation using your selection.

Press "Cancel" to abort this operation. No files will be changed.

Note that the information on top of the window shows you the real name that has caused the conflict, while that in the "rename" fields will be the new title of the object. &xwp; will automatically convert that title to a valid real name. In most cases however titles and real names do not differ.

You can re-enable the original &os2; dialog in the "&xwp; Setup" object, on the "Features" page.
